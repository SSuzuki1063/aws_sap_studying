AWS Organizations SCPの継承：超シンプル解説

SCPとは「サービスコントロールポリシー」のことで、AWSサービスの使用を制限するフィルターです。
この図解では、SCPがどのように継承されるかを超シンプルに説明します。

1. SCPの基本: フィルターの仕組み

SCPの基本ルール：SCPは許可を与えるものではなく、許可を「フィルター」するものです！

2. 階層構造での継承

組織ルート
基本的な制限を設定
↓
組織単位 (OU)
部門別の制限を追加
↓
アカウント
個別の制限を適用

3. 重要なポイント

  • SCPは制限のみを行い、許可は与えません
  • 上位のSCPで拒否されたアクションは、下位で許可できません
  • すべての階層のSCPが「許可」している場合のみ、アクションが可能です
  • IAMポリシーとは別物です - IAMは許可、SCPは制限

4. 実践的な例

シナリオ：開発チームのEC2制限

組織ルート： すべてのAWSサービスを許可

開発OU： EC2の本番環境インスタンスタイプを拒否

開発アカウント： 追加制限なし

結果： 開発アカウントでは小規模なEC2インスタンスのみ使用可能

AWS Direct Connect 仮想ゲートウェイ解説

AWS Direct Connectとは？

AWS Direct Connectは、お客様のデータセンターやオフィスからAWSへの専用ネットワーク接続を提供するサービスです。インターネットを経由せずに、より安全で高速かつ安定した接続を実現します。

主な利点

🚀 高性能

最大100Gbpsの専用帯域幅で、予測可能な性能を提供

🔒 セキュリティ

インターネットを経由しない専用回線で、データの安全性を確保

💰 コスト効率

大量のデータ転送において、インターネット経由よりもコスト削減

仮想ゲートウェイ (VGW) の役割

仮想ゲートウェイ（Virtual Gateway）は、VPCとDirect Connect接続の間の橋渡しを行うAWSマネージドコンポーネントです。

主な機能：

  • VPCとオンプレミス間のルーティング
  • BGPセッションの管理
  • 複数VPCへの接続（VPC Peering経由）
  • 冗長性の提供

設定手順

1
Direct Connect接続の申し込み

AWSコンソールからDirect Connect接続を申し込み

2
仮想インターフェース (VIF) の作成

パブリックVIFまたはプライベートVIFを選択

3
仮想ゲートウェイの作成・接続

VGWを作成し、VPCに接続

4
ルートテーブルの設定

オンプレミスへのルートを追加

🛡️ AWS WAF Web ACLルールモード

警備システムで理解するWebセキュリティ

AWS WAFとは？

AWS WAF（Web Application Firewall）は、あなたのWebアプリケーションを守るデジタル警備システムです。
建物に警備員がいるように、WebサイトにもWAFという警備員が必要です。

5つのルールモード

📊

COUNT モード

「記録する警備員」

怪しい人を見つけても止めずに、記録だけをつける警備員
🚫

BLOCK モード

「厳格な警備員」

怪しい人を見つけたら即座に入場拒否する警備員
✅

ALLOW モード

「親切な警備員」

特定の条件の人は必ず通してくれる警備員
🤖

CAPTCHA モード

「確認する警備員」

「あなたは人間ですか？」と質問する警備員
🔍

CHALLENGE モード

「調査する警備員」

JavaScript実行できるかテストする高度な警備員

どのモードを使うべき？

🔬 テスト段階

COUNT → ログを見て効果を確認

⚡ 緊急時

BLOCK → 攻撃を即座に停止

🎯 信頼できるユーザー

ALLOW → VIPユーザーの優遇

🤖 ボット対策

CAPTCHA/CHALLENGE → 自動化攻撃を防ぐ

💡 プロのヒント

  • まずはCOUNTで様子見
  • 効果が確認できたらBLOCKに変更
  • 重要なAPIにはCAPTCHAを設定
  • 管理者IPはALLOWで除外
