AWS Migration HubとApplication Discovery Agentの関係

クラウド移行の第一歩をわかりやすく解説

超シンプル解説：何をするものか？

AWS Migration Hub：クラウド移行の「司令塔」です。移行全体を一元管理する指揮センターのようなもの。

Application Discovery Agent：現在のサーバーの情報を収集する「調査員」です。どんなアプリが動いているか、どのくらいのリソースを使っているかを調べます。

両者の関係：調査員（Agent）が集めた情報を司令塔（Hub）に送ることで、最適な移行計画が立てられます。

身近な例えで理解する

引っ越しに例えると…

• 今の状況：あなたが今住んでいる家（オンプレミス環境）

• Application Discovery Agent：引っ越し業者の見積もり担当者。家の間取り、家具のサイズ、荷物の量などを調査する。

• AWS Migration Hub：引っ越しの総合窓口。見積もり担当者が集めた情報を基に、トラックの手配、作業員の割り当て、スケジュール管理などを一元的に行う。

• 目的地：新しい家（AWS環境）

AWS Migration HubとApplication Discovery Agentの関係図

AWS Migration Hub

クラウド移行の「司令塔」です。

簡単に言うと：
移行全体を見渡して管理するダッシュボード

何ができるの？

  • 一元管理：すべての移行作業を一か所で確認できる
  • 進捗確認：どこまで移行が進んだかを見える化
  • 移行方法選択：最適な移行方法を提案してくれる

例えるなら：
引っ越し会社の総合窓口

Application Discovery Agent

現在のサーバー情報を収集する「調査員」です。

簡単に言うと：
サーバーにインストールする小さなプログラム

何ができるの？

  • 情報収集：サーバーのスペックやアプリの情報を集める
  • 関係性調査：アプリ同士のつながりを調べる
  • 使用状況測定：CPUやメモリの使用量を記録する

例えるなら：
引っ越し前の家の調査をする担当者

移行の流れ：わずか6ステップ

1

準備

AWSコンソールにログインして、Migration Hubにアクセスします。Discovery Agentのダウンロードリンクを取得します。

簡単に言うと： 引っ越し業者に連絡して、調査の日程を決める

2

エージェントの設置

移行したいサーバーそれぞれにDiscovery Agentをインストールします。Windows、Linuxなど様々なOSに対応しています。

簡単に言うと： 調査員が各部屋の寸法を測る

3

情報収集

エージェントが自動的にサーバーの情報を集めて、AWS Migration Hubに送信します。収集した情報は暗号化されて安全に保管されます。

簡単に言うと： 家具のサイズや数、特殊な荷物があるかを調べる

4

データ分析

Migration Hubが集めた情報を分析して、最適な移行方法を判断するための材料を提供します。

簡単に言うと： 集めた情報を基に、必要なトラックのサイズや人員を計算する

5

移行計画作成

分析結果をもとに、最も効率的な移行計画を立てます。サーバーごとに最適な移行方法を選択できます。

簡単に言うと： 引っ越し当日のスケジュールと手順を決める

6

実行と監視

計画に沿って移行を実行し、Migration Hubで進捗状況をリアルタイムで確認します。

簡単に言うと： 引っ越し作業を進めながら、全体の進み具合をチェック

初心者向け用語解説

オンプレミス環境：自社内でサーバーを所有・管理している状態。自分の家に家具や家電を持っているようなもの。
クラウド移行：自社サーバーからAWSなどのクラウドサービスに移すこと。レンタル家具付きの家に引っ越すようなイメージ。
エージェント：特定の仕事を自動的に行うプログラム。この場合は情報収集の役割を担う。
リホスト：そのままの状態でサーバーを移行する方法。引っ越しで家具をそのまま新居に運ぶようなもの。
リプラットフォーム：一部を最適化して移行する方法。引っ越し時に一部の家具を新調するようなもの。

まとめ：初心者が押さえるべきポイント

  1. Migration Hubはクラウド移行全体を管理する「司令塔」
  2. Discovery Agentは現在のサーバー情報を集める「調査員」
  3. エージェントが集めた情報をHubに送ることで、効率的な移行計画が立てられる
  4. 手作業での調査が不要になり、ミスも減る
  5. まずは小さなサーバーから始めて、徐々に拡大するのがおすすめ

クラウド移行は大きなプロジェクトですが、この２つのツールを使えば、初心者でも安心して取り組めます！
