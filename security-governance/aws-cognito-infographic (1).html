AWS Cognito ユーザープールとIDプールの完全ガイド

アプリケーションの認証・認可を実現する総合ソリューション

AWS Cognitoは、ウェブおよびモバイルアプリケーション向けの包括的な認証・認可・ユーザー管理サービスです。主に「ユーザープール」と「IDプール（アイデンティティプール）」の2つのコンポーネントを提供し、これらを連携させることで、セキュアでスケーラブルなユーザー認証とAWSリソースへのアクセス管理を実現します。

Uユーザープール

ユーザープールは、ユーザーディレクトリであり、アプリのユーザー登録と認証機能を提供します。ユーザーの属性情報の管理、多要素認証、パスワードポリシーなどの設定が可能です。

✓主な機能:
  • ユーザー登録・サインイン
  • アカウント確認と回復
  • ユーザープロファイル管理
  • カスタム属性の定義
✓認証方法:
  • ユーザー名/パスワード
  • Eメール/電話番号
  • ソーシャルIDプロバイダー（Facebook, Google, Appleなど）
  • SAML/OIDCフェデレーション
  • カスタム認証フロー
✓セキュリティ機能:
  • 多要素認証（MFA）
  • 適応型認証（リスクベース）
  • 高度なパスワードポリシー
  • アカウントのロック機能
  • ブルートフォース対策
✓認証結果:

JWT（JSON Web Token）の発行:

  • IDトークン: ユーザー識別情報を含む
  • アクセストークン: 権限情報を含む
  • 更新トークン: 新しいトークンの取得に使用
✓主なユースケース:
  • Webアプリの認証基盤
  • モバイルアプリのユーザー管理
  • APIへのセキュアなアクセス
  • シングルサインオン（SSO）の実現

ユーザープールの作成

IIDプール（アイデンティティプール）

IDプールは、AWS リソースへのアクセス権を付与するためのものです。ユーザーの認証情報（ユーザープールのトークン、ソーシャルIDなど）を、AWS一時認証情報と交換する機能を提供します。

✓主な機能:
  • 一時的なAWS認証情報（IAMロール）の提供
  • 複数の認証プロバイダーのサポート
  • ロールベースのアクセス制御
  • 認証済み/未認証ユーザーの区別
✓認証プロバイダー:
  • Cognitoユーザープール
  • ソーシャルプロバイダー（Facebook, Google, Amazonなど）
  • OpenID Connect（OIDC）プロバイダー
  • SAML IDプロバイダー
  • デベロッパー認証プロバイダー（カスタム）
✓セキュリティ機能
